# 2.1 线性表的定义与基本操作
## 线性表的定义
- 线性表是具有**相同**（占的空间一样）数据类型的$n$个**数据元素**的**有限序列**（有次序），其中$n$为**表长**，当$n=0$时线性表为**空表**。用$L$来命名线性表，则一般表示为
$$L=(a_1,a_2,…,a_i,a_{i+1},…,a_n)$$
![线性表的形式](http://oss.pyaxy.xyz/img/%E7%BA%BF%E6%80%A7%E8%A1%A8%E7%9A%84%E5%BD%A2%E5%BC%8F.png)
>[!note]
>1. $a_i$是线性表中的“第i个”元素线性表中的**位序**
> 1. $a_1$是表头元素；$a_n$是表尾元素
> 2. 除第一个元素外，每个元素有且仅有一个**直接前驱**；除最后一个元素外，每个元素有且仅有一个**直接后继**
---
## 线性表的基本操作
``` cpp
// 命名方式参考：《数据结构》by 严蔚敏
InitList(&L) // 初始化表，分配内存空间
DestroyList(&L) // 销毁表，释放内存空间

ListInsert(&L, i, e) // 插入操作，在表L的第i个位置插入元素e
ListDelete(&L, i, &e) // 删除操作，删除表L的第i个元素e并返回。

LocateElem(L, e) // 按值查找，在表L中查找值为e的元素
GetElem(L, i) // 按位查找，获取表L中第i个元素的值

Length(L) // 求表长
PrintList(L) // 输出操作
Empty(L) // 判空操作
```

# 2.2 顺序表
- 顺序表一一用**顺序存储**的方式实现线性表
- 顺序存储：
	- 把**逻辑上相邻**的元素存储在**物理位置上也相邻**的存储单元中，元素之间的关系由存储单元的邻接关系来体现。
>[!info]
>- 随机存取：
>- 随机存取是指存/取元素的方式为随机存取，顺序表的存取方式为随机存取。==注意区分与顺序存储之间的关系==
---
## 顺序表的实现--静态分配
``` cpp
#define MaxSize 10 // 最大长度
typedef struct {
	ElemType data[MaxSize]; // 静态数组存储顺序表
	int length; // 顺序表的长度
} SqList;

void InitList(SqList &L) { // 初始化顺序表
  for (int i = 1; i < MaxSize; i++) {
    L.data[i] = 0;
  } // 防止脏数据
  L.length = 0; // 初始长度为0
}
```
>[!note]
>静态分配顺序表(使用数组语句声明)时，顺序表内部的数组内存会分配到**栈(stack)空间**中。栈区用于存储局部变量和函数调用有关的信息。当函数调用结束后，数组的生命周期也随之结束，即自动释放空间，而栈上分配的数组大小通常是**编译时已确定的，无法扩展**。
---
## 顺序表的实现--动态分配
``` cpp
#include <stdio.h>
#include <stdlib.h>
#define InitSize 10
#define ElemType int

typedef struct {
  ElemType *data; // 指针指向动态数组起点
  int MaxSize;    // 最大长度
  int length;     // 当前长度
} SqList;

void InitList(SqList &L) {
  L.data = (ElemType *)malloc(sizeof(ElemType) * InitSize); // 申请内存
  L.MaxSize = InitSize; // 初始最大空间为分配空间大小
  L.length = 0;         // 初始长度为0
}

void IncreaseSize(SqList &L, int len) {
  ElemType *p = L.data; // 辅助指针指向原数组位置
  L.data = (ElemType *)malloc(
      sizeof(ElemType) *
      (L.MaxSize + len)); // 为数组申请新的空间，大小为old + len
  for (int i = 0; i < L.MaxSize; i++)
    L.data[i] = p[i];          // 将旧数据转移到新位置
  L.MaxSize = L.MaxSize + len; // 修改最大长度
  free(p);                     // 释放原来空间
}
```
>[!note]
>采用`malloc`函数分配的内存位于**堆(heap)**区域内，需要手动释放内存空间(`free()`)，即`malloc`与`free`必须成对出现，其特点是可以在程序运行过程中根据需求动态的调整大小。
- 顺序表的特点
	1. 随机访问，可在$O(1)$的时间内访问第i个元素。
	2. 存储密度高，每个节点只存储数据元素。
	3. 拓展容量不方便。
	4. 插入删除不方便。
---
## 顺序表的插入
- ListInsert(&L, i, e): 插入操作。在表L的第i个位置插入元素e。
``` cpp
// 顺序表的插入
bool InsertList(SqList &L, int i, int e) {
  // 超出当前顺序表范围判为非法，可用范围为已用空间的任一个地方以及表尾插一个，共length+1个
  if (i < 1 || i > L.length + 1)
    return false;
  // 表已满判为非法
  if (L.length == L.MaxSize)
    return false;

  // 从后往前移动元素，不能从前往后，会产生覆盖
  for (int j = L.length; j >= i; j--)
    L.data[j] = L.data[j - 1];
  // 第i个位置的索引为i-1
  L.data[i - 1] = e;
  // 长度加1
  L.length++;
  return true;
}
```
- 上面插入函数时间复杂度为$O(n)$
---
## 顺序表的删除
``` cpp
// 顺序表的删除
bool DeleteList(SqList &L, int i, int &e) {
  // 判断i的范围，范围为1-length
  if (i < 1 || i > L.length)
    return false;
  // 返回删除的值到e里
  e = L.data[i - 1];
  // 从前往后删除
  for (int j = i; j < L.length; j++)
    L.data[j - 1] = L.data[j];
  // 长度-1
  L.length--;
  return true;
}
```
- 上面删除函数时间复杂度为$O(n)$
---
## 顺序表的查找
``` cpp
// 在顺序表中查找第一个值为 e 的元素，返回其位序
int LocateElem(SqList &L, int i, ElemType e) {
  for (int i = 0; i < L.length; i++) {
    if (L.data[i] == e)
      return i + 1;
  }
  return 0;
}
```
- 上面按值查找函数的时间复杂度为$O(n)$
``` cpp
// 顺序表中按位查找元素并返回
ElemType GetElem(SqList &L, int i, ElemType e) { 
  return L.data[i - 1];
}
```
- 上面按位查找函数的时间复杂度为$O(1)$

# 2.3 链表
## 单链表
### 单链表的定义
- 单链表中的各个节点在**物理上是离散存储**的
	- 优点：对大片空间需求少，扩容方便
	- 缺点：不支持随机存取
- 使用代码定义单链表结构体
``` C
typedef struct LNode{// 定义节点
	ElemType data;// 存入数据
	struct LNode *next;// 指向下一个节点
}LNode, *LinkList;
```
>[!note]
>注意上面将`struct LNode`重命名为`LNode`；将`struct LNode*`重命名为`LinkList`其主要是因为表达侧重点不一样：
>1. `LNode *N`强调N是一个节点。
>2. `LinkList L`强调L是一个链表。
>>[!warning]
>>虽然上面两者在物理机里都表示对结构体的指针，但是侧重点不一样，具体可以见[[线性表#单链表的查找（带头节点）|GetItem()函数]]中，函数传入的参数中有`LinkList &L`，其表示的重点为它为一个链表，在函数内部新建的节点指针时使用`LNode *`其表示的重点为它只是一个节点。
>>
---
#### 定义不带头节点的单链表
- 代码定义不带头节点的单链表
``` C
typedef struct LNode{
	ElemType data;
	struct LNode *next;
} LNode, *LinkList;

bool InitList(LinkList &L) {
	L = NULL;
	return true;
}
```
>[!note]
>不带头节点的单链表初始为空，所以为NULL，同样防止脏数据。
---
#### 定义带头节点的单链表
- 代码定义带头节点的单链表
``` C
typedef struct LNode{
	ElemType data;
	struct LNode *next;
} LNode, *LinkList;

bool InitList(LinkList &L) {
	// 申请头节点内存
	L = (LNode *)malloc(sizeof(LNode));
	if(L == NULL)
		return false;
	L->next = NULL;
	return true;
}
```
>[!note]
>带头节点的单链表初始化时存在一个节点，不存数据，表空所以指向的下一个节点为`NULL`。

>[!tip]
>简单评判一下是否带头节点：不带头节点的链表代码更复杂，需要处理特殊情况；带头节点的链表代码简单。
---
### 单链表的插入
- 按位序插入(带头节点)
``` C
bool ListInsert(LinkList &L, int i, ElemType e) {
  // 判断非法位置
  if (i < 1)
    return false;
  LNode *p = L; // p 指向当前遍历到的节点
  int j = 0; // j 代表当前p遍历到第几个节点，带头节点的链表头节点为第0个
  while (p != NULL && j < i - 1) { // 找到第 i - 1 个节点后停止
    p = p->next;
    j++;
  }
  // i 值不合法
  if (p == NULL)
    return false;
  
  // 申请新节点插入
  LNode *s = (LNode *)malloc(sizeof(LNode));
  if (s == NULL)
    return false;
  s->data = e;
  s->next = p->next;
  p->next = s;
  return true;
}
```
- 上面代码的时间复杂度为$O(1)$
- 重点掌握循环找到第 i - 1 个节点的遍历以及申请新节点的代码
---
- 按位序插入(不带头节点)
```C
bool ListInsert(LinkList &L, int i, ElemType e) {
  if (i < 0) {
    return false;
  }
  LNode *p = L;
  if (i == 1) { // 第一个节点需要特殊操作，因为头指针需要变更
    LNode *s = (LNode *)malloc(sizeof(LNode));
    s->data = e;
    s->next = L;
    L = s;
    return true;
  }
  int j = 1; // j 表示当前遍历到第 j 个节点，初始值为 1
  while (p != NULL && j < i - 1) { // 找到第 i 个节点的前一个节点
    p = p->next;
    j++;
  }
  if (p == NULL) { // 非法位置判断
    return false;
  }
  
  // 新建节点并插入
  LNode *s = (LNode *)malloc(sizeof(LNode));
  s->data = e;
  s->next = p->next;
  p->next = s;
  return true;
}
```
>[!info]
>如果不带头结点，则插入、删除第1个元素时，需要改变头指针L的指向，所以需要特殊操作；带头结点时，头指针一直指向头结点，因此不需要特殊操作。
---
- 指定结点的后插操作
```C
// 指定节点的后插操作
bool InsertNextNode(LNode *p, ElemType e) {
  if (p == NULL) {
    return false;
  }

  LNode *s = (LNode *)malloc(sizeof(LNode));
  if (s == NULL) { // 内存分配失败
    return false;
  }

  s->data = e;
  s->next = p->next; // s连接到p之后
  p->next = s;
  return true;
}

```
---
- 指定节点的前插操作
```C
// 指定节点的前插操作
bool InsertPriorNode(LNode *p, int e) {
  if (p == NULL)
    return false;
  LNode *s = (LNode *)malloc(sizeof(LNode));
  if (s == NULL)
    return false;

  // 核心思想，后插结点，交换数据
  s->next = p->next;
  p->next = s;
  s->data = p->data;
  p->data = e;
  return true;
}
```
>[!info]
>指定节点的前一个节点在未给出头指针的情况下是无法寻找到的，因此我们可以采用将新节点插入指定节点的后面，再交换两个节点的数据，即实现了新节点在指定节点的前面。

### 单链表的删除
- 按位序删除(带头节点)
```C
// 删除位序 i 处的节点，并把数据由 e 返回
bool ListDelete(LinkList &L, int i, ElemType &e) {
  // 非法位序
  if (i < 0)
    return false;
  LNode *p = L;
  int j = 0;
  // 找到第 i - 1 个节点
  while ( p != NULL && j < i - 1) {
    p = p->next;
    j++;
  }
  // i 值不合法
  if (p == NULL)
    return false;
  // 第 i - 1 个节点后已无其他节点
  if (p->next == NULL)
    return false;
  LNode *q = p->next; // q 指向被删除节点
  e = q->data; // 用 e 返回元素值
  p->next = q->next; // q节点从中间断开
  free(q); // 释放内存
  return true;
}
```
---
- 删除指定节点
```C
// 删除指定节点
bool DeleteNode(LNode *p) {
  if (p == NULL)
    return false;
  LNode *q = p->next; // q 指向 p 后面的节点
  p->data = q->data; // 后面节点的值向前覆盖
  p->next = q->next; // 将 q 节点从中间断开
  free(q); // 释放空间
  return true;
}
```
>[!info]
>在找不到前驱节点的时候还是采用操作数据位置的思想，删除节点p可以通过删除p的后继节点来实现，实质就是将其后继的值赋予其自身，然后删除后继。
>>[!warning]
>>但是上述代码在遇到p为最后一个节点时有bug，因为p的后继节点为NULL，后续对NULL的所有操作都不成立。

### 单链表的建立(带头节点)
- 尾插法
- 思想：设置一个尾指针r，每次对r指针的后面插入一个新节点，并更新r的位置
```C
// 尾插法建立链表/正向建立单链表
LinkList List_TailInsert(LinkList &L) {
  int x;
  // 建立头节点
  L = (LinkList)malloc(sizeof(LNode));
  // r 为尾指针
  LNode *r = L;
  // 输入节点的值
  scanf("%d", &x);
  while (x != -1) { // 输入 -1 表示结束
    LNode *s = (LNode*)malloc(sizeof(LNode));
    s->data = x;
    r->next = s;
    r = s; // r 指向新的表尾节点
    scanf("%d", &x);
  }
  r->next = NULL; // 最后一个节点指向NULL
  return L;
}
```
---
- 头插法
- 思想：每次都插入在头节点之后。
- 重要应用：链表的逆置
	- 循环扫描原链表的每一个元素，把每一个节点用头插法插入到头节点之后，即实现了链表的原地逆置
```C
// 头插法建立链表/反向建立链表
LinkList List_HeadInsert(LinkList &L) {
  int x;
  // 建立头节点
  L = (LinkList)malloc(sizeof(LNode));
  L->next = NULL; // 初始化为空链表
  scanf("%d", &x);
  while (x != -1) {
    LNode *s = (LNode*)malloc(sizeof(LNode));
    s->data = x;
    s->next = L->next;
    L->next = s;
    scanf("%d", &x);
  }
  return L;
}
```
>[!warning]
>注意上面代码中的`L->next = NULL`不可省略，因为新建的`L`中，`L->next`会指向未知脏数据，而后面又有`s->next = L->next`，若不初始化为`NULL`，第一个插入的节点(表尾节点)指向的下一个位置就不为`NULL`，而是未知的脏数据区域。
>为了避免这种情况发生，无论写什么代码时，每次新建链表时都初始化一下。

## 双链表
- 双链表有两个指针，一个指向前驱节点，一个指向后继节点
- ![](http://oss.pyaxy.xyz/img/20250525194632889.png)
- 结构定义
```C
typedef struct DNode{
	ElemType data;
	struct DNode *prior, *next;
}DNode, *DLinkList;
```
### 双链表的初始化
- 双链表的初始化(带头节点)
```C
// 双链表初始化
bool InitDLinkList(DLinkList &L) {
  // 分配一个头节点
  L = (DLinkList)malloc(sizeof(DNode));
  // 内存不足
  if (L == NULL)
    return false;
  // 头节点之后暂时还没有节点
  L->next = NULL;
  // 头节点的 prior 指针永远指向 NULL
  L->prior = NULL;
  return true;
}
```
---
### 双链表的插入
- 在节点后插入一个新节点
```C
// 在p节点后插入s节点
bool InsertNode(DNode *p, DNode *s) {
  if (p == NULL || s == NULL)
    return false;
  s->next = p->next;
  if (p->next != NULL){ // 如果p有后继节点
    p->next->prior = s;
  }
  s->prior = p;
  p->next = s;
  return true;
}
```
>[!info]
>1. 注意修改指针时要注意顺序，而且注意可能出现的bug，**如p后面没有节点时应该跳过对下一个节点的前驱的处理。**
>2. 在双链表中用后插操作可以方便的实现前插，对i节点进行前插即是对i节点前驱的后插，双链表可以很方便的找到前驱和后继。
---
### 双链表的删除
- 删除p节点的后继节点
```C
// 删除p节点的后继节点
bool DeleteNextNode(DNode *p){
  if (p == NULL)
    return false;
  DNode *q = p->next; // q 指向后继节点
  if (q == NULL) // 如果没有后继节点
    return false;
  p->next = q->next;
  if (q->next != NULL) // q 节点后面还有节点
    q->next->prior = p;
  free(q);
  return true;
}
```
- 销毁一个双链表
- 思想：循环删除头节点的下一个节点，直到头节点指向NULL，再释放头节点。
```C
void DestoryList(DLinkList &L){
  while (L->next != NULL)
    DeleteNextNode(L);
  free(L);
  L = NULL;
  return;
}
```

## 循环链表
### 循环单链表
- 最后一个节点的`next`指针指向头节点，从一个节点出发可以找到其他任何一个节点。
- ![](http://oss.pyaxy.xyz/img/20250525201609348.png)
- 初始化时头节点的`next`指针指向自己
- 一些操作
```C
// 初始化一个循环单链表
bool InitList(LinkList &L) {
  L = (LNode *)malloc(sizeof(LNode));
  if (L == NULL)
    return false;
  L->next = L; // 头节点next指向自己
  return true;
}

// 判断是否为空
bool Empty(LinkList &L) {
  return L->next == L;
}

// 判断是否为尾节点
bool isTail(LinkList L, LNode *p) {
  return p->next == L;
}
```
## 循环双链表
- 表头节点的`prior`指针指向表尾节点，表尾节点的`next`指针指向表头节点。
- ![](http://oss.pyaxy.xyz/img/20250525202335015.png)
- 一些操作
```C
// 初始化空的循环双链表
bool InitDLinkList(DLinkList &L) {
	L = (DNode *)malloc(sizeof(DNode));
	if (L == NULL)
		return false;
	L->prior = L; // 头节点的prior指向头节点
	L->next = L; // 头节点的next指向头节点
	return true;
}

// 判断是否为空
bool Empty(DLinkList &L) {
	return L->next == L;
}

// 判断是否为尾节点
bool isTail(DLinkList L, DNode *p) {
	return p->next == L;
}
```
---
>[!与双链表的对比]
>在[[线性表#双链表的插入|双链表的插入]]这部分代码中，特判了当p节点为最后一个节点时候的情况，但是在循环双链表中，我们不需要特判，直接一系列操作即可
- 循环双链表的插入
```C
bool InsertNextNode(DNode *p, DNode *s) {
	s->next = p->next;
	s->prior = p;
	p->next->prior = s;
	p->next = s;
}
```
---
>[!与双链表的对比]
>双链表的删除也是一样，与[[线性表#双链表的删除|双链表的删除]]这部分代码中的特判部分在循环双链表中也可以省略。
- 循环双链表的删除
```C
bool DeleteNextNode(DNode *p) {
	DNode *q = p->next;
	p->next = q->next;
	q->next->prior = p;
	free(q);
}
```

## 静态链表
- 分配一整片连续的内存空间，各个节点集中安置，以下标来确定位置，下标为0处为头节点，游标为-1表示到达表尾。
- ![](http://oss.pyaxy.xyz/img/20250526184329460.png)
- 结构
```C
#define MaxSize 10
typedef struct Node{
	ElemType data; // 数据
	int next; // 游标
} SLinkList[MaxSize];

void test() {
	SLinkList a; // 这里a为大小为MaxSize的数组
}
```
- 初始化
```C
// 初始化
void InitSLinkList(SLinkList &L) {
	L[0].next = -1;
}
```

- 查找
	- 从头节点出发挨个向后遍历

- 在位序为 i 处插入节点
	- 找到一个空的位置，比如初始化时，把空位置的游标设置为 -2
	- 从头节点出发找到 i - 1 节点
	- 修改新节点的next为i-1的next
	- 修改i-1的next

- 删除某个节点
	- 找到前驱节点
	- 修改前驱游标
	- 被删除节点游标改为 -2

## 顺序表与链表的对比

|                    | 顺序表                                                                     | 链表                                                                |
| ------------------ | -------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| 逻辑结构           | 线性表                                                                     | 线性表                                                              |
| 存储结构           | 顺序存储：支持随机存取；存储密度大；大片连续空间分配不便；改变容量不便     | 链式存储：空间分配方便；改变容量方便；不支持随机存取；存储密度低    |
| 基本操作（创建）   | 初始化时需要预分配大片连续空间。空间过大过小都不方便；                     | 初始化时只需分配一个头节点，之后便于扩展                            |
| 基本操作（销毁）   | 修改 `length = 0`，使用系统自动回收（静态数组）或者`free`（动态数组）      | 依次删除各个节点`free`                                              |
| 基本操作（增、删） | 插入/删除元素要将后续元素后移/前移，时间复杂度$O(n)$，时间开销在于移动元素 | 插入/删除只需要修改指针，时间复杂度$O(n)$，时间开销在于寻找目标元素 |
| 基本操作（查）     | 按位查找，$O(1)$；按值查找，$O(n)$                                         | 按位查找，$O(n)$；按值查找，$O(n)$                                  |

- 用顺序表还是链表：
	- 表长难以预估、经常要增加/删除元素：链表
	- 表长固定，经常查找元素：顺序表

|                | 顺序表 | 链表 |
|:--------------:|:------:|:----:|
| 弹性（可扩容） |   😭   |  😄  |
|     增、删     |   😭   |  😄  |
|       查       |   😄   |  😭  |
